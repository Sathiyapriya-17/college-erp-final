# 3. SYSTEM ANALYSIS

## 3.1. Overview of the Proposed System
The proposed College ERP System represents a paradigm shift from traditional, siloed administrative processes to a highly cohesive, digital ecosystem. Designed with a decoupled architecture, the system separates the presentation layer from the business logic. The frontend is engineered as a responsive Single Page Application (SPA) using Angular fortified with the PrimeNG component library, delivering a seamless, state-driven user experience without turbulent page reloads. The backend operates robustly on the Django framework integrated with the Django REST Framework (DRF), exposing a suite of secure, stateless RESTful APIs. Data persistence and relational integrity are guaranteed by PostgreSQL, a powerful open-source database system capable of handling complex academic schemas.

## 3.2. Functional Analysis
The system's modular architecture is designed to address specific, critical workflows within the institution:

*   **Role-Based Authentication & Security:** The system mandates secure entry points through differentiated login portals (`student-login/` and `faculty-login/`). It incorporates an automated user onboarding pipeline where administrative approvals of registration requests systematically trigger credential generation and email dispatch. Moreover, robust password recovery mechanisms are institutionalized via time-sensitive OTP resets.
*   **Comprehensive Academic Management:** The academic hierarchy is strictly modeled, encompassing Departments, Courses, and Curriculum components. The system allows administrators to dynamically assign subjects to specific faculty members, ensuring clear accountability and resource allocation per semester.
*   **Precision Attendance Tracking:** Moving away from error-prone registers, the system automates attendance logging. Faculty members record presence dynamically via specialized grid interfaces course-wise and date-wise. The backend APIs instantly calculate aggregate metrics, providing real-time attendance visualization on both student and faculty dashboards.
*   **End-to-End Examination Processing:** The examination module (`ExamViewSet`) facilitates the complete lifecycle of assessments. It enables faculty to schedule exams, link external digital assessments like Google Forms, and issue automated calendar alerts. Crucially, the system features an automated grading engine (`ExamAttemptViewSet`); upon the entry of line-by-line marks by faculty, the REST APIs programmatically trigger CGPA recalculations, eliminating human mathematical errors entirely.
*   **Dynamic Timetable Scheduling:** Complex institutional scheduling is resolved through interactive timetable visualization. The data structures map classes to specific time-slots, rooms, and faculty, providing editable administrative views alongside read-only graphical timetables for students.
*   **Centralized Communications and Fee Management:** The platform replaces physical notice boards with a digital announcement system, categorizing alerts by priority and target role (Student/Faculty). Concurrently, a dedicated Fee Management module tracks student financial dues, installment status, and payment histories transparently.

## 3.3. Technical and Performance Analysis
By leveraging an Angular-based SPA, the application significantly reduces server payload by rendering UI components client-side, making subsequent API calls exclusively for JSON data retrieval. The Django backend seamlessly handles heavy computational tasks such as bulk attendance metric aggregations and real-time dashboard analytics (e.g., enrollment trends and financial summaries displayed via Chart.js). The adoption of PostgreSQL ensures ACID compliance, which is mission-critical for maintaining the integrity of sensitive student academic grades and financial fee records. Ultimately, this decoupled approach ensures the platform is highly scalable, easily maintainable, and primed for future institutional expansion.
